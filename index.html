vim编辑器使用

发表于 2017-09-14 | 分类于 linux

简介

在1976年之前，UNIX系统中的标配编辑器并不是vi，而是ed，一种行编辑器。
Bill Joy一开始开发了对用户更友好、支持更多命令的ed——ex(ed extended)。
紧接着，他同Chuck Haley一起为ex开发了ex的visual interface，也就是后来的vi。
1979年，vi正式采用了vi这个名字，并沿用至今。

VIM有输入模式、编辑模式和末行模式。

一、权限管理

综述

文件的权限主要针对三类对象进行定义：
    owner: 属主, u
    group: 属组, g
    other: 其他, o
每个文件针对每类访问者都定义了三种权限：
    r: Readable
    w: Writable
    x: eXcutable
对于文件来说：
    r: 可使用文件查看类工具获取其内容；
    w: 可修改其内容；
    x: 可以把此文件提请内核启动为一个进程；
对于目录来说：
    r: 可以使用ls查看此目录中文件列表；
    w: 可在此目录中创建文件，也可删除此目录中的文件；
    x: 可以使用ls -l查看此目录中文件列表，可以cd进入此目录；    
权限标识:
    --- 000 0
    --x 001 1
    -w- 010 2
    -wx 011 3
    r-- 100 4
    r-x 101 5
    rw- 110 6
    rwx 111 7
例如：
    640: rw-r-----
    rwxr-xr-x: 755

权限修改

u表示属主，g表示属组，o表示其他人，a表示这三者。此命令root和普通用户都可用

chmod [OPTION]... MODE[,MODE]... FILE...
chmod [OPTION]... OCTAL-MODE FILE...
chmod [OPTION]... --reference=RFILE FILE...
    -R: change files and dir recursively 递归修改目录及子目录
#递归修改/tmp/man及其子目录权限为777    
chmod -R 777 /tmp/man   

#修改a.txt文件权限为属主rw-，属组不变，其他人为r--
chmod u=rw,o=r a.txt

#修改a.txt文件权限为属组rw-，属主和其他人权限不变
chmod g=rw- a.txt

#将a.txt文件的属组的w权限去掉，其他权限不变，+号同理
chmod g-w a.txt

#修改所有人的权限为r--
chmod a=r-- a.txt

#将a.txt文件的权限修改为/tmp/b.txt的权限
chmod --reference=/tmp/b.txt a.txt

修改文件属组和属主

仅限root账户可用。

chown [OPTION]... [OWNER][:[GROUP]] FILE...
chown [OPTION]... --reference=RFILE FILE...
    -R: 递归修改其自身及子目录

#修改a.txt属主为ygq    
chown ygq a.txt

#修改a.txt属主为ygq,属组为hadoop2
chown ygq:hadoop2 a.txt

#修改a.txt属组为root
chown :root a.txt

#修改a.txt属组和属主同a一样
chown --reference=a a.txt

umask

文件和目录创建时的遮罩码

#查看当前umask  默认为0022
umask  

#设置当前umask为3
umask 3

FILE: 666-umask
#如果某类的用户创建的文件权限减得的结果中存在x权限，则将其权限+1
#例如当umask为3，创建文件a  666-3 = 633 存在x权限，则变成644

DIR: 777-umask

二、grep、egrep、fgrep

基于正则表达式进行文本过滤的工具
根据用户指定的“模式”对目标文本逐行进行匹配检查，并能够打印匹配到的行

正则表达式

正则表达式分为基本正则表达式(BRE)和扩展正则表达式(ERE)

grep默认只能使用基本正则,egrep是使用扩展正则,fgrep不支持正则表达式搜索,但性能最好

字符匹配

. (点号)匹配任意单个字符

[] 匹配指定范围内的任意单个字符

[^]：匹配指定范围外的任意单个字符
[:digit:]任意数字，相当于0-9
[:lower:]任意小写字母
[:upper:]任意大写字母
[:alpha:]任意大小写字母
[:alnum:]任意数字或字母
[:space:]单个空格
[:punct:]标点符号

匹配次数

用在要指定次数的字符后面，用于指定前面的字符要出现的次数

*：匹配前面的字符任意次

.* : 任意长度的任意字符

下面的\代表转义,使用egrep或者 grep -E可以不用转义

?：匹配其前面的字符0或1次

+：匹配其前面的字符至少1次

{m}：匹配前面的字符m次；

{m,n}：匹配前面的字符至少m次，最多n次；

{0,n}：匹配前面的字符最多n次；

{m,}：匹配前面的字符至少m次；

位置锚定

^：行首锚定；用于模式的最左侧

$：行尾锚定；用于模式的最右侧

\< 或 \b：词首锚定 用于单词模式的左侧；

必须加\转义

词尾锚定 用于单词模式的右侧；

必须加\转义

匹配整个单词

必须加\转义

()分组

分组主要用来后向引用，即后面引用前面匹配到的字符,-E可以不用加\转义
分组括号中的模式匹配到会被记录为 \1, \2, \3…
(ab+(xy))：
\1: ab+(xy)
\2: xy

|或者

拓展表达式才有
C|cat: C或cat
(C|c)at : Cat或cat

I/O重定向

  1. 输出重定向

2>>: 追加重定向错误输出数据流
ls -l >> /tmp/success.out 2> /tmp/error.out
&>：覆盖重定向错误和正确输出数据流
&>>：追加重定向错误和正确输出数据流
ls -l &> /dev/null
  2. 输入重定向
    tr a A < /etc/fstab     #将/etc/fstable文件中所有a替换成A输出到屏幕

文本处理工具

  1. tr

    Translate,and/or delete characters from std input, writing to std output.

    tr [OPTION]... SET1 [SET2]  
tr [:lower:] [:upper:] < /etc/fstab #将/etc/fstab文件小写换大写
tr 'a-z' 'A-Z' < /etc/fstab #将/etc/fstab文件小写换大写
tr -d fs < /etc/fstab    #将/etc/fstab文件中所有f s字符删除展示

  2. tee

    read from standard input and write to standard output and files

    tee [OPTION]... [FILE]...
echo "$PATH" | tr 'a-z' 'A-Z' | tee /tmp/a.txt
echo "$PATH" | tee /tmp/a.txt | tr 'a-z' 'A-Z'
#tee默认覆盖输出
tee -a, --append #append to the given FILEs, do not overwrite

  3. wc

    print newline, word, and byte counts for each file

    wc [OPTION]... [FILE]...
    -m, --chars
      print the character counts

    -l, --lines
            print the newline counts

    -L, --max-line-length
            print the length of the longest line

    -w, --words
            print the word counts

  4. cut

    逐行按指定分隔符进行切割，并按指定分隔符将指定位置拼合起来输出

    cut OPTION... [FILE]...
-d DELIMITER: 指明分隔符
-f FILEDS:
    #: 第#个字段
    #,#[,#]：离散的多个字段，例如1,3,6
    #-#：连续的多个字段, 例如1-6
    混合使用：1-3,7
--output-delimiter=STRING
    use STRING as the output delimiter 
    the default is to use the input delimiter
cat /etc/passwd | cut -d':' -f1,2-4
    root:x:0:0
    bin:x:1:1
    daemon:x:2:2
cat /etc/passwd | cut -d':' -f1,2-4 --output-delimiter=#
    root#x#0#0
    bin#x#1#1
    daemon#x#2#2

  5. sort

    对指定输入进行排序

    sort [OPTION]... [FILE]...
    -f: 忽略字符大小写
    -r: 逆序排序
    -t DELIMITER: 字段分隔符
    -k #：以指定字段为标准排序
    -n: 以数值大小进行排序
    -u: uniq，排序后去重

#以:为分隔符，对/etc/passwd的第3字段忽略大小写逆序排序并去重
sort -t: -k3 -fru /etc/passwd

  6. uniq命令：

    能统计每行重复出现的次数

    uniq [OPTION]... [FILE]...
    -c: 显示每行重复出现的次数；
    -d: 仅显示重复过的行；
    -u: 仅显示不曾重复的行；    
history | cut -d' ' -f5 | sort | uniq -c

useradd [options] 用户名

6. linux安全上下文

进程所能够访问的所有资源的权限取决于进程的发起者的身份

像cat等等的命令，对于linux来说只是普通的应用程序
当用户输入`cat /etc/fstab`时，cat命令是以当前shell进程登录用户的权限去访问的
linux系统上每一个文件都有权限和属主和属组的
比如一个文件权限为rwxrw-r--
那么当某用户已cat命令访问这个文件时，linux系统会进行如下判断：
如果该用户是此文件属主，那么则可以rwx权限访问
如果该用户不是属主，但是属组中的一员，则以rw-权限访问
如果既不是属主又不是属组，则以r--权限访问

  1. *任意长度的任意字符

 ab, a12b, abc 不符合

  5. 专用字符集合

        # cp -a /etc/m*[^0-9] /tmp/mageedu

        # cp -a /etc/*.d /tmp/magedu.com

        # cp -a /etc/[mnrp]*.conf  /tmp/mageedu.com

linux文件系统

  1. 说明
    /是linux系统中所有文件访问的起始点
linux中所有文件要想被访问到，必须和/建立关联
系统启动时，系统会将/挂载至某一分区上，即根分区
对于linux系统，我们可以创建多个分区，并且可以手动将不同的分区关联到不同的路径上
比如系统中有A、B两个分区，A分区已经挂载到了/
那么我们访问/mnt就会去A分区的某个磁道上面去找mnt目录
如果我们将/mnt挂载到B分区，那么当我们访问/mnt/test.txt时，
就会先去A分区找到/，然后找到A分区上面的mnt目录，然后会去B分区找test.txt文件
  2. 分区的好处
    如果我们不进行分区，所以的文件都集中在/所在的分区也可以
但是这样想要重新格式化文件系统的时候就不得不重装系统
如果我们分为多个分区，某分区格式化时不会影响别的分区

  3. 各目录作用

关于终端

linux系统中，一切皆文件，哪怕是设备。终端设备也在文件中有所体现

  1. 控制台终端(console):

    早期由于计算机特别贵，而且unix是多人多任务操作系统，
所以为了能让多人同时使用unix进行工作,所以就采用比较便宜的物理终端接入unix
控制台终端所使用文件/dev/console

  2. 虚拟终端(tty)：

    linux系统启动时，会自动启动6个虚拟终端
这6个虚拟终端之间可以使用ctrl+alt+F1-6之间切换
使用tty查看当前使用的终端。
tty
#/dev/tty1
#输出结果和当前使用的是哪个终端相关，比如ctrl+alt+F2,则显示/dev/tty2,其他同理

  3. 伪终端(pts):

    一般通过ssh远程连接登录的终端就属于伪终端，使用tty命令查看：
#/dev/pts/1
#如果同时再有一个ssh连接到本系统，则显示/dev/pts/2
#ls /dev/pts/   此命令可查看当前连入系统的有哪几个虚拟终端

关于命令

linux系统中有两种命令，内建命令、外部命令，使用type命令可查看命令类型

  1. 内建命令：
    由shell自带
  2. 外部命令
    当前系统的文件系统某路径下有对应执行文件
which和whereis可查看所在路径
shell按照PATH环境变量所指定的路径自左向右的依次搜寻
echo $PATH 查看PATH环境变量
只有第一次才会去搜寻路径，一旦搜寻到之后，会缓存至一个内存中的key-value存储中
使用hash可查看该缓存
hits    command         #hits为缓存命中次数
5        /bin/ls
如果hash中缓存的程序路径发生改变，比如mv /bin/ls /usr/bin/ls，则会找不到ls命令
此时可以使用hash -r 清空缓存
  3. 命令行接口： 
    [root@hostname ~]# command [options] [arguments]
    root : 当前登录的用户
    hostname : 当前主机名，显示的并非完整主机名
    ~ : 当前处于的目录，也称工作目录，此处显示的是相对路径，使用pwd命令可显示完整目录
    # : 命令提示符，其中#代表root用户；$代表普通用户 
    options: 有短选项如 -l -h -c等，短选项一般可合并，如-lhc
                有长选项如 --help等，长选项一般不能合并
    arguments: 命令的作用对象，多个参数之间一般用空格隔开        
    ctrl+c可取消命令执行

history命令

linux系统中，所有命令历史都保存在内存中。当用户登出时，系统会将内存中的命令历史追加写入当前用户家目录的.bash_history文件中

  1. 常用选项
    history -a      #追加本次登录后使用的命令至.bash_history文件中
history -d 40   #删除指定编号的历史命令
history -c      #清空命令历史
history 10      #显示历史中最近的10条命令；
  2. 快捷操作
    !40             #调用历史中指定编号的命令
!str            #调用历史中最近一次包含str的命令
!!              #调用上一次命令
ESC然后按.       #调用上次命令的最后一个参数
  3. 环境变量

man命令

  1. 简介

    man手册所在目录为：/usr/share/man  
不同作用的命令存在不同的目录
man命令会去对应的目录下找到.gz格式的压缩文件进行解压并用less命令展示结果
有些命令存在于多个章节中，whatis <command>可以查看该命令在哪个章节
使用 man num <command> 可以查看第num章节的帮助文档

  2. 章节介绍

    man1            #user commands   普通用户命令  
man2            #system call     系统调用
man3            #c库调用
man4            #设备文件及特殊文件
man5            #配置文件格式
man6            #游戏相关
man7            #杂项
man8            #管理类命令

  3. man命令配置文件

    配置文件：       /etc/man.config  这里面指明去哪搜索man文档

  4. man命令段落说明

    NAME            #功能性说明
SYNOPSIS        #语法格式
                []：可选内容；
                <>：必须提供的内容；
                a|b|c：多选一；
                ...：同类内容可出现多个；
DESCRIPTION     #描述
OPTIONS         #选项
EXAMPLES        #示例
  5. 操作方法
    空格键           #向文件尾翻一屏；
b               #向文件首部翻一屏；
Ctrl+d          #向文件尾部翻半屏；
Ctrl+u          #向文件首部翻半屏；
回车键           #向文件尾部翻一行；
k               #向文件首部翻一行；
G               #跳转至最后一行；
2G              #跳转至第2行；

  6. 文本搜索

    /keyword        #从文件首部向文件尾部依次查找；不区分字符大小写；
?keyword        #从文件尾部向文件首部依次查找；    
n               #与查找命令方向相同；
N               #与查找命令方向相反；

  7. 退出

    q               #quit
